## 二、核心设计原则（沿用设计文档）

1. **具体合约，不搞合成** — 存 `RB2510` 这类真实可交易合约，主力连续 `RB0` 仅用于辅助标记，不作为主数据。
2. **品种 / 合约两层结构** — `products`（品种层，如 RB、IO、600019.SH）→ `symbols`（合约层，如 RB2510、IO2509-C-3700）。
3. **统一多周期 K 线表** — 所有资产、所有周期共用一张 `kline` 表，用 `timeframe` 字段区分。
4. **聚合优先于重拉** — M15/M30/H1/H4 由 M5 聚合；W1/M1 由 D1 聚合，避免重复请求接口。
5. **主键天然去重** — `kline` 主键 `(datetime, symbol_id, timeframe)`，重复拉取不产生重复行。

## 四、数据库表结构（9 张表）

DuckDB 文件: `data_center/data_center.db`。建表 SQL: `data_center/db/init_schema.sql`。

| 表名 | 作用 | 主键 |
|:----|:----|:----|
| `products` | 品种层（RB/IO/600019.SH 等） | product_id |
| `symbols` | 合约层（RB2510 等，含期权行权价/到期日） | symbol_id |
| `kline` | 统一多周期 K 线（所有资产共用） | (datetime, symbol_id, timeframe) |
| `options_daily` | 期权日线 IV/希腊值/理论价 | (date, symbol_id) |
| `stocks_info` | 股票基本信息（PE/PB/市值/行业） | symbol_id |
| `stocks_financial` | 股票财报（营收/净利/ROE 等） | (symbol_id, report_date) |
| `futures_main_switches` | 主力合约换月记录 | switch_id |
| `macro_data` | 宏观经济指标时间序列 | (date, product_id) |
| `cross_market_mapping` | 跨市场映射 + 相关系数 | mapping_id |

### 关键表字段速查

**products** — code, name, asset_type(futures/stock/option/index/macro), exchange, sector, multiplier, tick_size, margin_rate …

**symbols** — product_id(FK), code, contract_year, contract_month, is_main, option_type(call/put), strike_price, underlying_id, list_date, expire_date …

**kline** — datetime, symbol_id(FK), timeframe(M5/M15/M30/H1/H4/D1/W1/M1), open, high, low, close, volume, amount, open_interest, settlement, pre_settlement

**options_daily** — date, symbol_id, iv, iv_percentile, hv_30, delta, gamma, theta, vega, rho, theoretical_price, time_value, intrinsic_value, underlying_close

**cross_market_mapping** — product_id_a, product_id_b, relation_type(成本传导/利润联动/上下游), logic_desc, corr_20d, corr_60d, lead_lag_corr, lag_days

## 八、已知限制与注意事项

1. **DuckDB 单进程独占锁** — 同一时刻只能有一个进程打开 `data_center.db`。因此**全量下载脚本与后端服务不能同时写库**。一次性历史下载用独立脚本跑；日常使用时下载应改为后端进程内的后台任务。

2. **期权希腊值是实时快照** — `option_risk_analysis_em` 返回当日盘中快照，非交易时段为空。历史希腊值需逐日累积采集。

3. **TqSdk 免费账户仅近期数据** — 深度历史用 akshare/Tushare 回补，TqSdk 主要用于 tick / 实时。

4. **前端既有问题（与本次无关）** — `frontend/src/pages/Strategy.tsx:257` 有一处 TS 类型告警（number/string），属历史遗留，未改动。

5. **网络容错** — Tushare fetcher 内置指数退避重试，应对网络维护期的 ECONNRESET 中断；下载脚本断点续传，中断可恢复。

## 十、2026-06-19 增量升级

在 06-18 统一仓库基础上，围绕「生产可运维」做了一轮深度升级。核心是**合约生命周期**与**按年同步面板**。

### 10.1 合约生命周期 (`data_center/knowledge/contract_lifecycle.py`)

期货/期权统一处理。每个真实合约有生命周期（挂牌→交割），生命周期外不应有数据。

- `parse_expiry(code)` — 按交割月月末解析到期日（M2609 → 2026-09-30）。
- `status(code)` — 合约状态：**在挂**（今天≤到期）/ **已到期**（今天>到期）/ **连续**（纯品种无固定到期）。
- `lifecycle_window(code)` — 有效数据窗口 [到期-3年, 到期]。
- `lifecycle_guard(df, code)` — 入库前裁剪窗口外的脏数据。

> 解决的真实问题：手动下载 `symbol=M + contract=M2609` 时，数据源忽略 contract 返回主力连续 M 的全历史（2005 起），被误存成 M2609。守卫裁掉了 4567 条生命周期外数据。

### 10.2 按年同步面板（前端「实时同步」tab）

一年一行倒序（当前年→2015），每行三列（期货/股票/期权），每格：

- **状态**：未同步 / 已同步 / 同步中（带 checkpoint 进度）。状态推导自仓库实有数据（期货期权按 `contract_year`，股票按 K线自然年）。
- **数量**：期货期权 `有数据/总合约`，股票 `N 只`。
- **操作**：同步（后台任务，断点续传）/ 校验（条数/NaN/空合约质量检查）。

实时增量同步（轮询调度器）整合在同一面板。这样**全量历史同步**与**实时同步**在一个模块内完成。

### 10.3 商品期权按年逐日全量采集

`OptionsCollector.collect_commodity_year(year)`：遍历该年每个交易日 × DCE/CZCE/SHFE 全品种，拉当日全合约 → 覆盖该年所有挂过的合约。

- **三种合约代码格式统一解析**（`SymbolRegistry.parse_contract` 扩展）：
  - DCE 连字符：`m2608-C-2500`
  - CZCE 3位年月：`SR609C4600`（6=2026, 09月）
  - SHFE 4位年月：`cu2607C76000`
- **IV/Delta 交易所直供**：CZCE/DCE 提供隐含波动率+Delta，SHFE 提供 Delta，直接落 `options_daily`，无需自算。
- 缺 IV 的（SHFE）可用 Black76 (`options_analytics.compute_option_greeks`) 补算全套希腊值。
- 断点续传（ckpt_done 集合）防限流/中断。

> 期权按年覆盖能力分层：**商品期权**（代码含年月）可真正按年逐日全量；**股指期权**（IO/HO）代码含年月、按 contract_year 归档但新浪仅给当前在挂；**ETF 期权**（510050 等流水号）按到期日归年、仅当前在挂。

### 10.4 知识库三库并行

- **合约知识库**（已有）+ 结构化字段扩充：宏观敏感度、季节性、核心基本面指标、关联品种。
- **股票知识库**（新）：申万行业板块 + 行业↔期货联动映射（钢铁→RB/I/J 等）。
- **期权知识库**（新）：标的市场特征（IV区间/流动性/行权方式）+ 期权策略库。

> 注：评分引擎、季节性、持仓情绪、跨市场相关、IV/技术指标等**信号计算能力本仓库已存在**（resonance/analysis/options/signals），本次只补「描述性知识层」，未重复造轮子。

### 10.5 存储与运维

- **两套存储分工**（明确）：手动下载走 **Parquet 单文件**（`data/market/...`，快速试探/导出）；生产数据以 **DuckDB 仓库**为准（采集/全量/同步）。两者独立不自动同步。
- **存储管理**：单文件删除 + 导出 `.xlsx`；列表聚合期货/股票/期权三类。
- **概览 8 卡**：数据源、期货品种、股票/期权品种数、交易所、K线总条数、**数据库物理大小**（实时监控物理空间）。

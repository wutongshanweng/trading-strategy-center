## 目标

**一个完整的仪表盘页面**，让交易者在一个页面看完"市场发生了什么 + 宏观趋势 + 对我做的品种有什么影响 + 未来怎么看"。

不拆分一期二期，一次做完。消息无需推送，页面本身 1 小时自动刷新。

## 模块详细设计

### 1. 实时快讯 — `news/fetchers/cls.py`

**数据源：财联社电报 (cls.cn)**
- 接口：`https://www.cls.cn/telegraph` (免费可访问)
- 抓取最新 50-100 条快讯
- 按品种关键词自动标标签：螺纹钢/RB/铁矿石/铜/原油/黄金/农产品 等
- 情绪标记：关键词正面→🟢/负面→🔴/中性→🟡
  - 利多词：突破/大涨/增/利好/回升/扩张/超预期
  - 利空词：大跌/降/利空/回落/收缩/不及预期/制裁

**刷新策略**：页面 1h 自动刷新，后端缓存最近 200 条

**存储**：`data/news_cache.json`（简单 JSON 持久化，重启保留）

### 2. 宏观指标看板 — `macro/aggregator.py`

**复用已有 `macro_collector.py`**：
- 调用 akshare 采集 CPI/PPI/PMI/GDP/M2/LPR
- 已落入 DB `macro_data` 表，直接查最新值 + 前值
- 计算变化方向（↑/↓/持平）和幅度
- API: `GET /api/v1/macro/dashboard` → 各指标最新值+趋势+更新时间

### 3. 宏观事件日历 — `news/calendar.py`

**数据源：静态日历 + 财联社抓取**
- 内置固定事件：美联储议息/非农/中国PMI/CPI发布日/两会 等
- 财联社快讯中提取"即将发布/重点关注"类信息
- API: `GET /api/v1/calendar?days=14`

**日历种子数据（内置）：**
```
美联储: 每年8次议息会议（1/3/5/6/7/9/11/12月）
中国: PMI 每月末、CPI/PPI 每月9号、GDP 每季次月
非农: 每月第一个周五
```

### 4. 联动分析 — `macro/regime_adapter.py` + `api/routes/macro_routes.py`

**核心逻辑（新写）：**

```
输入: 新闻情绪分 + 宏观指标变化 + 当前市态
输出: 品种影响预测 + 市态判断
```

**实现方式（初期规则引擎，不依赖 ML）：**
- 品种关键词→新闻映射（静态规则表）
- 宏观指标与品种的预设关联度（固定，非机器学习）：
  ```
  螺纹钢 ← PMI(+), M2(+), 房地产新开工
  铜     ← PMI(+), GDP(+), 全球制造业
  原油   ← 地缘政治, OPEC, 库存
  农产品   ← CPI, 天气, 进出口
  ```
- 市态判断：宏观指标组合→趋势/震荡/高风险

### 5. 远期趋势展望

**基于已有的共振引擎 + 策略库的轻量推理：**
- 宏观指标多期趋势方向（连续上升/下降）
- 匹配历史类似宏观组合→品种表现（简易模式匹配）
- 输出自然语言解读（模板化拼接）

**第一期实现：**
- 连续 N 期宏观指标方向 → 预测文字模板
- 例如：`{指标}连续{期数}期{方向} → {品种}{预期}`

### 6. API 路由

```python
# api/routes/macro_news_routes.py  (单一路由文件)

GET    /api/v1/macro-news/dashboard     → 全量数据聚合（新闻+宏观+日历+联动+趋势）
GET    /api/v1/macro-news/news          → 新闻列表（分页/按品种过滤）
GET    /api/v1/macro-news/macro         → 宏观指标看板
GET    /api/v1/macro-news/calendar      → 宏观日历
GET    /api/v1/macro-news/linkage       → 联动分析
GET    /api/v1/macro-news/outlook       → 远期趋势
```

设计为一个 `/dashboard` 接口返回全部数据，前端一次请求渲染整页。

### 7. 前端 — `frontend/src/pages/MacroNews.tsx`

**单一页面，5 个区域垂直布局：**

| 区域 | 组件 | 数据 |
|------|------|------|
| 🔔 实时快讯 | 时间线列表 + 品种标签 + 情绪色标 | `dashboard.news` |
| 📊 宏观指标 | 6 卡片网格（最新值+趋势箭头+图表） | `dashboard.macro` |
| 📅 宏观日历 | 事件时间线（7-14天） | `dashboard.calendar` |
| 🔗 联动分析 | 市态描述 + 品种影响列表 + 关联度表格 | `dashboard.linkage` |
| 📈 远期趋势 | 趋势卡片+预测文本 | `dashboard.outlook` |

**自动刷新**：`useEffect` 挂载后立即加载，然后 `setInterval(fetch, 3600000)`

**菜单项**：`{ key: "/macro-news", icon: <FundOutlined />, label: "新闻宏观" }`

### 8. 后台定时采集（服务端定时刷新数据）

设一个简单的后台线程或定时任务来更新新闻和宏观数据（非前端定时器，而是后端缓存刷新）：

```python
# 启动时启动后台刷新线程
import threading

def _background_refresh():
    """每 30 分钟刷新新闻+宏观数据到缓存"""
    while True:
        time.sleep(1800)
        refresh_news_cache()
        refresh_macro_data()
```

或利用现有的 cron 机制：`main.py` lifespan 启动 `BackgroundTasks`

### 信号聚合器 — `signals/alert_aggregator.py`

```python
class AlertAggregator:
    """定时从各引擎收集信号，聚合去重排序，产出最终交易提醒列表。"""

    def collect(self, symbols: List[str]) -> List[AlertSignal]:
        """对关注的品种列表执行一次全扫描"""

    def run_once(self) -> List[AlertSignal]:
        """一次完整信号收集周期（定时任务调用）"""

    def get_active(self, limit: int = 10) -> List[AlertSignal]:
        """获取当前活跃信号（最新 N 条）"""
```

**扫描流程：**
1. 对每个关注的品种执行 `StrategyEngine.compute_all()`
2. 调用 `ResonanceEngine` 得到综合信号
3. 调用 `FactorAdvisor` 得到因子建议
4. 调用 ML 模型得到预测方向
5. 聚合各信号源，去重，按置信度/综合性排序
6. 给每个信号计算综合评星（策略数 + 因子数 + ML置信度 + 共振匹配度）
7. 存入 `data/alert_signals.json`（持久化）

**关注的品种列表：** 从 `core/config/settings.py` 读取或写一个静态配置

| ③ WebSocket | 接入券商/第三方实时流 | 高 |

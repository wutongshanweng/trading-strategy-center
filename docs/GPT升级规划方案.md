# Trading Strategy Center 升级规划方案

版本：v1.0  
定位：期货 + 期权 + 多模型策略共振的量化策略中心  
目标：把当前系统从“策略与模型集合”升级为“可复现、可风控、可组合、可扩展的衍生品策略平台”。

---

## 1. 总体升级目标

当前系统已经具备数据中心、策略库、Alpha101、ML、RL、期权、回测、风控、Web 前端等模块雏形。下一步不建议继续无序堆功能，而应围绕一条主线升级：

```text
市场数据
 -> 合约知识
 -> 特征工程
 -> 模型训练
 -> 策略信号
 -> 策略共振
 -> 组合构建
 -> 风控过滤
 -> 回测验证
 -> 模拟交易
 -> 监控归因
```

核心原则：

1. 模型可以多，但输入、输出、评估标准必须统一。
2. 策略可以多，但每个策略必须能被回测、归因、风控和组合。
3. 期货和期权不能当股票处理，必须引入合约乘数、保证金、换月、Greeks、波动率曲面、期限结构。
4. 不追求单个“神策略”，而追求策略簇、模型簇、品种簇之间的稳定共振。
5. 系统最终要回答三个问题：现在适合做什么、能下多大仓位、错了怎么退。

------

## 2. 三阶段升级路线

### 第一阶段：可信闭环

目标：先让一个期货模型闭环和一个期权模型闭环跑通。

期货闭环：

```
主力合约数据
 -> OHLCV + OI + 合约信息
 -> 特征工程
 -> LightGBM/XGBoost/HMM/GARCH 基线模型
 -> 方向/波动率/状态信号
 -> 期货规则回测
 -> 风控报告
```

期权闭环：

```
期权链
 -> IV / Greeks / Skew / Term Structure
 -> 期权策略信号
 -> 多腿组合
 -> Greeks 聚合
 -> 多腿回测
 -> 压力测试
```

第一阶段要完成的设计重点：

1. 统一 `InstrumentSpec`：品种、交易所、合约乘数、最小跳动、手续费、保证金、交易时间。
2. 统一 `MarketBar`：open、high、low、close、volume、open_interest、turnover、symbol、contract、datetime。
3. 统一 `FeatureFrame`：所有模型只读标准特征表。
4. 统一 `Signal`：方向、置信度、周期、来源模型、失效时间、建议仓位、风险标签。
5. 统一 `BacktestResult`：收益、回撤、夏普、卡玛、胜率、盈亏比、换手、滑点、最大连续亏损。
6. 统一 `RiskVerdict`：允许/拒绝、原因、最大仓位、止损、止盈、风险暴露。

第二阶段是系统的核心扩张期。目标不是简单增加策略数量，而是建立“策略家族 + 模型家族 + 共振引擎 + 组合分配”的完整体系。

## 4. 第二阶段重点策略设计

### 4.1 趋势跟踪策略

适用：强趋势、基本面驱动、资金推动、波动率扩张行情。

推荐策略：

1. 双均线趋势：EMA20/EMA60、EMA60/EMA120。
2. Donchian 通道：20 日突破入场，10 日反向突破离场。
3. Turtle 海龟策略：ATR 定仓，突破加仓，ATR 止损。
4. SuperTrend 策略：ATR 通道跟踪趋势。
5. ADX 趋势强度策略：ADX 过滤震荡，DI 判断方向。
6. KAMA 自适应均线策略：减少震荡期噪音。
7. 多周期趋势共振：日线定方向，小时线找入场。
8. 趋势斜率策略：线性回归斜率 + R2 过滤。
9. 价格通道趋势：布林上轨突破 + 波动率放大。
10. 趋势回踩策略：大周期趋势中，小周期回调到均线附近进场。

设计要点：

- 趋势策略不能单独使用，必须配合波动率和市场状态过滤。
- 趋势策略最怕震荡，要设计“震荡识别器”。
- 趋势策略应该允许小亏多次，靠少数大行情赚钱。

------

### 4.2 均值回归策略

适用：震荡、区间、低波动、无明显基本面驱动行情。

推荐策略：

1. Bollinger Band 回归：触及上下轨后回归中轨。
2. RSI 超买超卖：RSI < 30 做多，RSI > 70 做空。
3. Z-Score 偏离回归：价格偏离均值 N 个标准差。
4. VWAP 回归：日内价格偏离 VWAP 后回归。
5. Keltner Channel 回归：ATR 通道均值回归。
6. Pair Spread 回归：相关品种价差 Z-Score 回归。
7. OI 过热回归：价格上涨但持仓异常扩张后的回撤。
8. 期限价差回归：近远月价差偏离历史分位后回归。
9. 日内开盘缺口回补：高开/低开后缺口回补。
10. 季节性均值回归：农产品、能源化工适用。

设计要点：

- 均值回归必须有止损，不能用“越跌越买”替代风险管理。
- 最好配合市场状态：只在低趋势强度、低波动扩张时启用。
- 对期货要注意涨跌停和流动性风险。

------

### 4.3 突破策略

适用：盘整结束、波动率压缩后扩张、重大事件前后。

推荐策略：

1. 箱体突破：N 日高低点形成区间，放量突破。
2. 波动率压缩突破：ATR/布林带宽度处于低分位后突破。
3. 开盘区间突破：开盘前 15/30/60 分钟区间突破。
4. 前高前低突破：关键结构位突破。
5. 缺口突破：跳空后不回补，顺势跟随。
6. 成交量确认突破：突破同时 volume 或 turnover 放大。
7. 持仓确认突破：价格突破同时 OI 增加。
8. 多周期突破：小时级突破服从日线趋势。
9. 假突破反转：突破失败后反向入场。
10. 新闻事件突破：宏观数据、库存、政策发布后突破。

设计要点：

- 突破策略必须处理假突破。
- 入场可以激进，出场必须纪律化。
- 突破策略和趋势策略可以共振，但不能重复计算风险暴露。

------

### 4.4 动量策略

适用：强资金流入、板块轮动、连续上涨/下跌。

推荐策略：

1. 时间序列动量：过去 N 日收益为正则做多。
2. 横截面动量：同板块内强者恒强。
3. 板块轮动：黑色、有色、化工、农产品、能源之间轮动。
4. 收益加速度：收益率一阶、二阶变化。
5. 成交量动量：量价齐升。
6. 持仓动量：价格与 OI 同向变化。
7. 资金流动量：成交额、主力资金代理变量。
8. 多品种排名动量：选前 20% 做多、后 20% 做空。
9. 风险调整动量：收益 / 波动率。
10. 残差动量：剔除板块收益后的个体动量。

设计要点：

- 动量策略适合组合化，不适合只看单品种。
- 要控制品种集中度和板块集中度。
- 动量和趋势不同：趋势更关注路径，动量更关注相对强弱。

------

### 4.5 波动率策略

适用：期权、仓位管理、风险切换。

推荐策略：

1. 历史波动率突破：Realized Vol 上升，降低趋势仓位或跟随突破。
2. GARCH 波动率预测：预测未来波动，动态调仓。
3. HAR-RV 模型：日/周/月 realized volatility 建模。
4. IV Rank 策略：IV Rank 高卖波动，低买波动。
5. IV-HV 溢价策略：隐波显著高于实现波动时卖波动。
6. Skew 策略：偏度过高/过低时构建价差。
7. Term Structure 策略：近月 IV 和远月 IV 结构异常。
8. Vol Carry 策略：收取波动率风险溢价。
9. Gamma Scalping：买期权后用期货动态对冲。
10. 波动率状态切换：低波趋势、高波趋势、高波震荡分别使用不同策略。

设计要点：

- 卖波动不是捡钱，是卖保险。
- 所有卖方策略必须有最大亏损、保证金、尾部压力测试。
- 波动率策略最适合作为策略共振中的“风险温度计”。

------

### 4.6 期限结构策略

适用：商品期货、股指期货、利率期货。

推荐策略：

1. Contango/Backwardation 判断。
2. 近远月价差趋势。
3. 月间价差均值回归。
4. 展期收益策略。
5. 主力换月价差策略。
6. 库存驱动期限结构策略。
7. 供需周期期限结构策略。
8. 跨月蝶式价差。
9. 期限曲线斜率策略。
10. 期限曲线曲率策略。

设计要点：

- 商品期货很多 Alpha 不在价格本身，而在期限结构。
- 期限结构策略对数据质量要求很高，要保证合约映射准确。
- 换月逻辑必须统一，不能每个策略自己处理。

------

### 4.7 跨期与跨品种套利

推荐策略：

1. 螺纹-热卷价差。
2. 焦煤-焦炭-铁矿黑色链价差。
3. 豆粕-豆油-大豆压榨利润。
4. 棕榈油-豆油价差。
5. PTA-乙二醇-短纤化工链价差。
6. 铜铝锌有色相对强弱。
7. 金银比价。
8. 股指 IF/IH/IC/IM 相对价值。
9. 国债期货跨品种曲线策略。
10. 内外盘价差：仅在数据可靠后加入。

设计要点：

- 套利不是无风险，核心风险是价差结构失效。
- 应该建协整检验、半衰期、相关性稳定性、价差 VaR。
- 跨品种策略必须有组合保证金和双腿成交滑点模拟。

------

### 4.8 期权策略库

推荐基础策略：

1. 买入 Call：方向看多 + IV 不贵。
2. 买入 Put：方向看空 + IV 不贵。
3. 卖出 Covered Call：持有期货多头时增强收益。
4. Protective Put：保护现有多头。
5. Bull Call Spread：温和看多，控制成本。
6. Bear Put Spread：温和看空，控制成本。
7. Short Straddle：高 IV、预期震荡。
8. Short Strangle：高 IV、宽区间震荡。
9. Long Straddle：低 IV、预期大波动。
10. Long Strangle：低 IV、预期方向不明但大波动。
11. Iron Condor：区间震荡 + 高 IV。
12. Butterfly：目标价格附近收敛。
13. Calendar Spread：期限结构异常。
14. Diagonal Spread：方向 + 波动率 + 时间结构。
15. Delta Neutral Vol Carry：中性卖波动。
16. Gamma Scalping：买 Gamma，动态对冲。
17. Skew Reversion：偏度回归。
18. Tail Hedge：极端风险保护。
19. Ratio Spread：非对称收益结构。
20. Synthetic Future：期权合成期货。

期权策略必须输出：

```
legs
net_premium
max_profit
max_loss
breakeven
delta
gamma
theta
vega
margin_required
iv_rank
stress_loss
```

### 4.9 机器学习策略

推荐模型任务：

1. 方向分类：未来 N 根 K 线涨跌。
2. 收益回归：未来 N 日收益率。
3. 波动率预测：未来 realized volatility。
4. 状态识别：趋势、震荡、高波、低波。
5. 策略选择：当前最适合哪个策略族。
6. 仓位建议：风险预算下的目标仓位。
7. 异常检测：行情、成交、信号异常。
8. 组合权重：多策略权重动态分配。

推荐模型：

1. Logistic Regression：强基线。
2. Random Forest：非线性基线。
3. XGBoost / LightGBM：主力表格模型。
4. HMM：市场状态识别。
5. GARCH / HAR-RV：波动率预测。
6. LSTM / TFT / N-BEATS：时间序列增强。
7. AutoEncoder：异常检测。
8. Reinforcement Learning：只用于执行、仓位或策略选择，不建议直接裸下单。
9. Stacking Ensemble：多模型融合。
10. Bayesian Model Averaging：模型置信度融合。

机器学习必须避免的问题：

- 未来函数。
- 数据泄漏。
- 随机切分时间序列。
- 只看收益不看回撤。
- 训练集表现很好，实盘全失效。
- 特征在回测存在，但实盘不可获得。

## 5. 策略共振设计

策略共振不是简单投票，而是多层确认。

建议共振结构：

```
Layer 1: 市场状态
Layer 2: 策略族信号
Layer 3: 模型预测
Layer 4: 跨品种/期限结构确认
Layer 5: 风险温度
Layer 6: 仓位分配
```

共振评分：

```
final_score =
  regime_score * 0.20
+ strategy_score * 0.25
+ model_score * 0.20
+ cross_asset_score * 0.10
+ volatility_score * 0.10
+ risk_score * 0.15
```

信号等级：

```
score >= 80: 强信号，可正常仓位
65 <= score < 80: 中信号，半仓或轻仓
50 <= score < 65: 观察信号，不开仓或试探
score < 50: 忽略
```

共振设计原则：

1. 趋势策略和均值回归策略不能在同一状态下同时满权重。
2. 波动率高时，仓位权重要自动下降。
3. 多个策略来自同一类因子时，要降相关性权重。
4. 期权卖方策略遇到高 Gamma 风险、低流动性、事件日前后必须降权。
5. 大模型生成的策略只能进入研究池，不能绕过回测和风控。

------

## 6. 策略组合设计

组合层目标：从“选一个策略”升级为“配置一篮子策略”。

组合方法：

1. 等权组合：基线。
2. 波动率倒数加权：降低高波策略权重。
3. 风险平价：每个策略贡献相近风险。
4. HRP：适合多策略相关性不稳定场景。
5. CVaR 最小化：控制尾部风险。
6. Kelly Fraction：只作为参考，实际使用分数 Kelly。
7. Regime-aware Allocation：不同市场状态使用不同权重。
8. Drawdown-aware Allocation：策略回撤扩大时自动降权。
9. Tournament Allocation：策略锦标赛排名决定资金分配。
10. Meta-model Allocation：用模型预测策略未来表现。

组合风控指标：

```
strategy_exposure
symbol_exposure
sector_exposure
directional_exposure
volatility_exposure
delta_exposure
vega_exposure
margin_usage
liquidity_score
correlation_cluster
```

## 7. 第三阶段：生产级加固

第三阶段目标：把系统从研究平台升级为可长期运行的平台。

### 7.1 数据加固

必须建立数据质量系统：

1. 缺失检测：K 线缺口、期权链缺失、持仓量缺失。
2. 延迟检测：实时行情延迟、数据源卡死。
3. 异常检测：价格跳变、成交量异常、负价格、重复数据。
4. 主力合约校验：换月日期、成交量/OI 切换规则。
5. 复权与连续合约：前复权、后复权、价差调整。
6. 多数据源交叉验证：AKShare、TDX、交易所、第三方数据。
7. 数据版本化：回测必须能知道用的是哪一版数据。
8. 数据血缘：每个特征能追溯到原始数据。

验收标准：

```
任何回测结果都必须能追溯：
数据源 -> 清洗规则 -> 特征版本 -> 模型版本 -> 策略版本 -> 参数版本
```

------

### 7.2 回测加固

回测必须分两套：

1. 向量化回测：用于快速筛选。
2. 事件驱动回测：用于真实评估。

事件驱动回测必须支持：

1. 合约乘数。
2. 保证金。
3. 手续费。
4. 平今手续费。
5. 滑点。
6. 冲击成本。
7. 涨跌停。
8. 夜盘。
9. 换月。
10. 多腿期权。
11. 部分成交。
12. 强平。
13. 资金占用。
14. 组合风险。
15. 交易日历。

回测报告必须包含：

```
收益类：total_return, annual_return, monthly_return
风险类：max_drawdown, volatility, VaR, CVaR
质量类：sharpe, sortino, calmar
交易类：win_rate, profit_factor, turnover, avg_holding_period
稳定性：walk_forward, rolling_sharpe, yearly_return
成本类：commission, slippage, impact_cost
归因类：direction_pnl, carry_pnl, vol_pnl, theta_pnl, cost_drag
```

------

### 7.3 风控加固

风控升级为四层：

```
订单前风控
组合风控
模型风控
系统风控
```

订单前风控：

1. 是否交易时段。
2. 是否涨跌停。
3. 是否流动性不足。
4. 是否超过单笔最大手数。
5. 是否超过品种最大仓位。
6. 是否保证金不足。
7. 是否临近交割。
8. 是否事件风险窗口。

组合风控：

1. 总杠杆。
2. 最大保证金占用。
3. 单品种集中度。
4. 单板块集中度。
5. 净方向敞口。
6. Delta / Vega / Gamma 暴露。
7. VaR / CVaR。
8. 最大回撤。
9. 连续亏损。
10. 相关性聚集。

2. 特征漂移。
3. 预测分布漂移。
4. 命中率劣化。

2. Redis/PostgreSQL 异常。
3. Celery 任务堆积。
4. API 错误率升高。
5. WebSocket 中断。

## 8. 我的交易系统设计观点

1. 好系统不是预测最准，而是错的时候亏得少，对的时候能拿住。
2. 策略越多，越需要统一评价标准，否则只是复杂度幻觉。
3. 期货的核心不是价格，而是杠杆、保证金、期限结构、换月和流动性。
4. 期权的核心不是涨跌，而是波动率、时间价值、尾部风险和 Greeks 暴露。
5. 趋势策略赚大行情的钱，均值回归赚噪音的钱，波动率策略赚风险溢价的钱，套利策略赚结构偏差的钱。
6. 策略共振的本质不是投票，而是不同信息源在同一方向上降低不确定性。
7. 单策略容易失效，策略族更稳定；单模型容易过拟合，模型组合更稳。
8. 回测收益不重要，回测可信度更重要。
9. 大模型可以扩大策略想象力，但不能替代验证系统。
10. 真正能长期运行的系统，靠的是数据质量、风控纪律、模型治理和持续归因。

# 6. 期货策略库设计

## 6.1 趋势跟踪策略族

适用行情：趋势明显、资金推动、波动率扩张、基本面驱动。

### 6.1.1 双均线趋势策略

逻辑：

```
EMA_fast > EMA_slow 做多
EMA_fast < EMA_slow 做空
```

推荐参数：

```
fast: [10, 20, 30]
slow: [50, 60, 120]
filter:
  adx_min: 20
  atr_percentile_min: 40
```

适用品种：

```
黑色、有色、能源、股指、贵金属
```

风险：

```
震荡行情频繁止损
```

改进：

```
加入 ADX 过滤、波动率过滤、多周期确认
```

### 6.1.2 海龟突破策略

逻辑：

```
突破 N 日高点做多
突破 N 日低点做空
ATR 定仓
ATR 止损
盈利后金字塔加仓
```

推荐参数：

```
entry_window: [20, 55]
exit_window: [10, 20]
atr_period: 20
risk_per_trade: 0.005-0.02
max_units: 4
```

适用：

```
趋势行情、商品大周期行情
```

失败模式：

```
假突破、横盘震荡、低流动性滑点
```

### 6.1.3 Donchian 通道策略

逻辑：

```
价格突破过去 N 日最高价 -> 多
价格跌破过去 N 日最低价 -> 空
```

增强条件：

```
成交量放大
持仓量同向增加
大周期方向一致
```

### 6.1.4 SuperTrend 策略

逻辑：

```
基于 ATR 计算上下轨
价格在上轨上方持多
价格在下轨下方持空
```

优势：

```
趋势跟踪清晰，止损自然
```

适合：

```
日线、小时线趋势策略
```

### 6.1.5 ADX 趋势过滤策略

逻辑：

```
ADX > 25 才允许趋势策略开仓
+DI > -DI 做多
-DI > +DI 做空
```

定位：

```
不是单独策略，而是趋势策略过滤器
```

### 6.1.6 KAMA 自适应均线策略

优势：

```
行情顺畅时快速跟随
噪音行情中自动变慢
```

适合：

```
有色、贵金属、股指
```

### 6.1.7 多周期趋势共振策略

逻辑：

```
日线判断方向
小时线寻找入场
15分钟线优化止损
```

共振规则：

```
日线多 + 小时线多 + 波动率未过热 -> 强多
日线多 + 小时线空 -> 等待
日线空 + 小时线空 -> 强空
```

### 6.1.8 趋势斜率策略

逻辑：

```
对过去 N 根 close 做线性回归
斜率 > 阈值且 R2 高 -> 趋势有效
```

参数：

```
window: [30, 60, 120]
min_slope_zscore: 1.0
min_r2: 0.3
```

------

## 6.2 均值回归策略族

适用行情：低趋势强度、震荡区间、波动率收缩。

### 6.2.1 Bollinger Band 回归

逻辑：

```
价格跌破下轨且 RSI 低 -> 做多
价格突破上轨且 RSI 高 -> 做空
回到中轨止盈
```

参数：

```
window: 20
std: [2.0, 2.5, 3.0]
rsi_filter: true
```

失败模式：

```
趋势行情中越跌越买、越涨越空
```

必须风控：

```
ADX > 25 禁用
突破后继续放量禁用
```

### 6.2.2 Z-Score 回归

逻辑：

```
zscore = (price - mean) / std
zscore < -2 做多
zscore > 2 做空
zscore 回到 0 平仓
```

适合：

```
价差、跨期、跨品种、低趋势品种
```

### 6.2.3 VWAP 日内回归

逻辑：

```
价格显著偏离 VWAP
若成交量不支持突破，则回归 VWAP
```

适合：

```
日内策略、股指期货、流动性好的商品
```

### 6.2.4 RSI 反转

逻辑：

```
RSI < 25 且价格未创新低 -> 做多
RSI > 75 且价格未创新高 -> 做空
```

增强：

```
配合背离检测
配合支撑阻力
```

### 6.2.5 价差均值回归

用于：

```
跨期价差
跨品种价差
产业链价差
```

核心指标：

```
价差 Z-Score
半衰期
协整检验
相关性稳定度
```

### 6.3.1 箱体突破

### 6.3.3 开盘区间突破

逻辑：

```

```

适合：

```

```

### 6.3.4 假突破反转

逻辑：

```
价格突破关键位后快速回落
成交量不能确认
反向开仓
```

适合：

```
震荡行情、压力支撑位
```

------

## 6.4 动量策略族

适用行情：资金连续流入、板块轮动、趋势延续。

### 6.4.1 时间序列动量

逻辑：

```
过去 N 日收益为正 -> 做多
过去 N 日收益为负 -> 做空
```

增强：

```
收益 / 波动率
收益排名
持仓量确认
```

### 6.4.2 横截面动量

逻辑：

```
同一板块内，对所有品种计算过去 N 日风险调整收益
做多排名前 20%
做空排名后 20%
```

适合：

```
黑色链、有色、化工、农产品、股指
```

### 6.4.3 板块轮动策略

逻辑：

```
比较板块强弱：
黑色、有色、能源、化工、农产品、贵金属、股指
资金流入强的板块提高权重
资金流出弱的板块降低权重
```

### 6.4.4 残差动量

逻辑：

```
先用板块指数解释品种收益
对残差做动量排名
```

优势：

```
剔除板块 Beta，捕捉个体 Alpha
```

------

## 6.5 期限结构策略族

适用市场：商品期货、股指期货、国债期货。

### 6.5.1 Contango / Backwardation 策略

逻辑：

```
近月 < 远月：Contango
近月 > 远月：Backwardation
```

应用：

```
Backwardation 通常代表现货紧张，多头更强
Contango 通常代表供给宽松或持仓成本高
```

策略：

```
Backwardation + 趋势向上 -> 多头加权
Contango + 趋势向下 -> 空头加权
```

### 6.5.2 跨期价差趋势

逻辑：

```
近月 - 远月 价差向上突破 -> 做多近月做空远月
近月 - 远月 价差向下突破 -> 做空近月做多远月
```

### 6.5.3 跨期价差回归

逻辑：

```
价差偏离历史均值 2 个标准差
等待回归
```

必须加入：

```
交割月过滤
流动性过滤
保证金双腿计算
```

### 6.5.4 期限曲线斜率策略

逻辑：

```
用多个合约构建期限曲线
计算斜率和曲率
根据曲线异常做套利
```

适合：

```
原油、金属、农产品、国债
```

# 7. 期权策略库设计

期权策略库必须围绕三个核心：

```
方向
波动率
时间价值
```

期权策略不能只输出买卖方向，必须输出多腿结构、最大亏损、保证金、Greeks。

------

## 7.1 单腿方向策略

### 7.1.1 买入 Call

适用：

```
方向强烈看多
IV 不贵
预期上涨速度快
```

入场条件：

```
趋势共振看多
IV Rank < 40
Delta 0.4-0.6
到期 20-60 天
```

风险：

```
Theta 损耗
方向对但涨太慢也亏
```

### 7.1.2 买入 Put

适用：

```
方向看空
波动率偏低
预期下跌速度快
```

入场条件：

```
趋势共振看空
IV Rank < 40
Delta -0.4 到 -0.6
```

------

## 7.2 价差策略

### 7.2.1 Bull Call Spread

结构：

```
买入低行权价 Call
卖出高行权价 Call
```

适用：

```
温和看多
IV 中等
想控制权利金成本
```

优势：

```
最大亏损有限
最大收益明确
```

### 7.2.2 Bear Put Spread

结构：

```
买入高行权价 Put
卖出低行权价 Put
```

适用：

```
温和看空
控制成本
```

### 7.2.3 Bull Put Spread

结构：

```
卖出高行权价 Put
买入低行权价 Put
```

适用：

```
不看跌
IV 偏高
希望收取权利金
```

风险：

```
尾部下跌风险
保证金占用
```

### 7.2.4 Bear Call Spread

结构：

```
卖出低行权价 Call
买入高行权价 Call
```

适用：

```
不看涨
IV 偏高
```

------

## 7.3 波动率策略

### 7.3.1 Long Straddle

结构：

```
买入同一行权价 Call
买入同一行权价 Put
```

适用：

```
IV 低
预期大波动
方向不确定
重大事件前
```

风险：

```
事件后 IV Crush
Theta 损耗
```

### 7.3.2 Short Straddle

结构：

```
卖出同一行权价 Call
卖出同一行权价 Put
```

适用：

```
IV 高
预期震荡
无重大事件
```

必须风控：

```
最大亏损压力测试
Delta 动态对冲
Gamma 风险监控
保证金充足
```

### 7.3.3 Long Strangle

结构：

```
买入虚值 Call
买入虚值 Put
```

适用：

```
低 IV
预期大波动但不确定方向
```

### 7.3.4 Short Strangle

结构：

```
卖出虚值 Call
卖出虚值 Put
```

适用：

```
高 IV
预期区间震荡
```

风险：

```
尾部风险极大
必须限制仓位
```

### 7.3.5 Iron Condor

结构：

```
卖出虚值 Put
买入更虚值 Put
卖出虚值 Call
买入更虚值 Call
```

适用：

```
高 IV
区间震荡
希望限制最大亏损
```

------

## 7.4 时间结构策略

### 7.4.1 Calendar Spread

结构：

```
卖出近月期权
买入远月同执行价期权
```

适用：

```
近月 IV 高
远月 IV 相对低
预期短期震荡
```

### 7.4.2 Diagonal Spread

结构：

```
不同到期日
不同行权价
```

适用：

```
方向 + 时间价值 + 波动率结构共同判断
```

## 7.5 Greeks 策略

### 7.5.1 Delta Neutral

目标：

```
组合 Delta 接近 0
主要赚波动率或时间价值
```

适用：

```
卖波动
Gamma Scalping
市场中性组合
```

### 7.5.2 Gamma Scalping

结构：

```
买入期权获得正 Gamma
用期货动态对冲 Delta
```

适用：

```
实际波动率高于隐含波动率
```

核心：

```
赚波动
亏 Theta
```

### 7.5.3 Vega 策略

逻辑：

```
IV 低买 Vega
IV 高卖 Vega
```

风险：

```
IV 可以长期低或长期高
不能只看历史分位
```

模型：

XGBoost
LightGBM
CatBoost
MLP

LSTM

标签设计：

```
未来 1 日收益

# 9. 大模型策略工厂

你后面使用多个大模型生成策略，这是很好的方向，但必须工程化。

## 9.1 大模型策略生成流程

```
LLM 生成策略想法
-> 策略标准化描述
-> 数据需求分析
-> 风险点分析
-> 伪代码生成
-> 自动代码审查
-> 静态规则检查
-> 小样本快速回测
-> 长周期回测
-> Walk-forward 验证
-> 参数稳定性分析
-> 策略相关性分析
-> 加入候选池
-> 模拟盘观察
-> 小资金实盘或继续淘汰
```

## 9.2 多模型协作分工

建议你让不同大模型承担不同角色：

```
模型 A：策略创意生成
模型 B：策略风险审查
模型 C：代码实现审查
模型 D：回测结果解释
模型 E：参数优化建议
模型 F：组合相关性分析
```

不要让一个模型既当作者又当审稿人。
策略生成和策略审查必须分离。

## 9.3 策略进入研究池条件

一个大模型生成策略必须满足：

```
有明确交易逻辑
有明确适用行情
有明确失效条件
有明确数据需求
有明确入场规则
有明确出场规则
有明确止损规则
有明确仓位规则
没有未来函数
没有不可获得数据
可以被回测
可以被风控
```

## 9.4 策略晋级流程

```
idea
-> draft
-> backtest_passed
-> walkforward_passed
-> paper_trading
-> candidate_live
-> production
-> degraded
-> retired
```

# 10. 策略共振系统设计

策略共振不是简单投票，而是不同信息源对同一方向的确认。

## 10.1 共振层级

```
Layer 1: 市场状态共振
Layer 2: 趋势/均值/突破等策略族共振
Layer 3: 机器学习模型共振
Layer 4: 跨品种与期限结构共振
Layer 5: 波动率与期权市场共振
Layer 6: 风控与仓位共振
```

## 10.2 共振评分公式

```
final_score =
  regime_score * 0.15
+ strategy_family_score * 0.20
+ model_score * 0.20
+ cross_asset_score * 0.10
+ volatility_score * 0.15
+ risk_score * 0.15
+ execution_score * 0.05
```

## 10.3 信号等级

```
score >= 85: A级强信号
75-85: B级有效信号
60-75: C级观察信号
45-60: D级弱信号
<45: 忽略
```

## 10.4 共振示例：期货做多

```
日线趋势向上
小时线突破
ADX > 25
OI 增加
成交量放大
期限结构 Backwardation
模型预测未来收益为正
波动率未过热
组合风险允许
```

满足越多，信号等级越高。

## 10.5 共振示例：卖出期权波动率

```
IV Rank > 80
实际波动率下降
标的处于震荡
无重大事件
Skew 未极端
Gamma 风险可控
保证金占用合理
压力测试亏损可接受
```

```
收益相关性
信号相关性
持仓方向相关性
回撤相关性
尾部亏损相关性
品种暴露相关性
```

两个策略如果都在同一时间亏钱，它们就不是有效分散。

策略之间要有不同收益来源
```

# 13. 交易理念和系统设计观点

## 13.1 关于策略

1. 不存在永远有效的策略，只存在适合某类行情的策略。
2. 趋势策略赚大行情的钱，均值回归赚噪音的钱，套利策略赚结构偏差的钱，期权策略赚波动率和时间价值的钱。
3. 策略越多，越要分类，否则策略库会变成垃圾堆。
4. 一个策略必须知道自己什么时候不该交易。
5. 策略失效不可怕，可怕的是系统不知道它失效了。

## 13.2 关于模型

1. 模型不是用来预测未来，而是用来降低不确定性。
2. 简单模型如果稳定，价值往往高于复杂模型。
3. 机器学习最容易犯的错是学到噪音。
4. 深度学习不应作为第一优先级，除非数据量、特征质量、验证体系都足够。
5. 大模型适合生成想法和解释，不适合直接下单。

## 13.3 关于期货

1. 期货交易的核心不是价格，而是杠杆、保证金、换月、流动性和风险暴露。
2. 商品期货很多机会来自期限结构、库存、供需和产业链。
3. 日线信号要考虑夜盘，夜盘经常改变风险结构。
4. 主力合约切换处理不好，回测结果会严重失真。
5. 期货策略必须尊重涨跌停和跳空风险。

## 13.4 关于期权

1. 期权不是方向工具，而是风险结构工具。
2. 买期权买的是速度和波动，卖期权卖的是保险。
3. 卖波动看起来胜率高，但真正风险在尾部。
4. Greeks 是期权风控语言，没有 Greeks 的期权系统不完整。
5. 期权策略必须明确最大亏损、保证金和压力场景。

## 13.5 关于组合

1. 多策略不是天然分散，低相关才是分散。
2. 策略组合的目标是穿越不同市场状态。
3. 单一高收益策略不如多个低相关中等收益策略。
4. 回撤控制比收益最大化更重要。
5. 组合系统要能自动降低失效策略权重。

# 14. 最终验收标准

系统升级成功后，应满足：

```
新增一个期货品种，不需要修改策略核心
新增一个期权合约，不需要修改风控核心
新增一个策略，可以自动注册、回测、评估、归类
新增一个模型，可以进入模型注册中心
任何策略信号，都能解释来源、逻辑、风险和历史表现
任何回测结果，都能追溯数据、参数、模型和代码版本
任何交易建议，都必须经过风控
任何大模型生成策略，都必须经过验证流程
策略库可以支持共振评分
策略组合可以动态分配资金
```

# 16. 一句话总结

Trading Strategy Center 后续最重要的方向不是继续堆模型，而是建立一个能够吸收模型、审查模型、组合模型、约束模型、淘汰模型的系统。
策略库要丰富，但必须被统一标准管理；大模型可以提供大量策略想法，但必须进入严格的研究、回测、风控和模拟盘流程。
真正成熟的交易系统，不是预测最准，而是能在不同市场状态下知道该用什么策略、用多少仓位、承担什么风险、什么时候退出。
